NEW REQUIREMENTS – MENU AVAILABILITY REAL-TIME BEHAVIOR

Read completely before coding.

Current project:
Cafe Maestro

# Problem 1

Admin can toggle menu item availability using is_available.

Current behavior:

Admin:
Cold Coffee
Available → OFF

QR Ordering:
Customer still sees old state until page refresh.

This is incorrect.

Required behavior:

Admin:
Available → OFF

QR Ordering:
Updates automatically without refresh.

Implementation requirements:

1. Use Supabase Realtime.

2. Subscribe to menu_items changes.

3. React immediately to:

   * INSERT
   * UPDATE
   * DELETE

4. Any change to:

   * is_available
   * name
   * price
   * image_url
   * description

must update customer view instantly.

No page refresh.

# Problem 2

Unavailable item already exists inside cart.

Scenario:

Customer cart:

Cold Coffee
Veg Burger

Admin:

Cold Coffee → OFF

Current cart:

Cold Coffee
Veg Burger

Required behavior:

Cart immediately re-validates.

Result:

Cold Coffee
→ marked unavailable

Veg Burger
→ remains normal

Show:

Out of Stock

Customer can still place order.

When placing order:

Unavailable items:

* removed from bill
* removed from order payload
* not charged

Remaining items:

* proceed normally

Show:

Out of Stock

# Problem 4

Server-side validation.

Even if customer never receives realtime event:

When order is submitted:

Backend must validate:

menu_items.is_available = true

before creating order.

Unavailable items must never reach kitchen.

This validation must exist server-side.

Client-side only validation is not sufficient.
