Feature: Advanced Table Management Dashboard

Goal:

Upgrade the Tables page into a real operational dashboard with live table status tracking and consistent state management.

IMPORTANT:

Before implementing UI changes, audit the existing data model and ensure there is a single source of truth for table state.

A table must never simultaneously appear as:

* Free
* Busy
* Booked
* Under Maintenance
* Archived

Only one state may be active at a time.

Create a status model:

ACTIVE STATES:

* free
* busy
* booked
* maintenance

SEPARATE FLAG:

* archived

Archived tables must never appear in operational views.

---

TABLE DASHBOARD CARDS

Replace:

* Total Tables
* Active
* Archived

With:

* Total Tables
* Free Tables
* Busy Tables
* Booked Tables
* Under Maintenance
* Archived Tables

All counts must update in real time.

Definitions:

FREE:

* active table
* no active session
* no active booking occupying it now
* not under maintenance

BUSY:

* active table
* active table session exists

BOOKED:

* confirmed booking exists for current date/time window

UNDER MAINTENANCE:

* maintenance toggle enabled

ARCHIVED:

* archived flag enabled

Free Tables:

Busy Tables:
→ show only busy tables

Booked Tables:
→ show only booked tables

Maintenance:

UNDER MAINTENANCE

Add maintenance toggle to each table.

ON:

* table hidden from booking assignment
* table hidden from available seating
* QR ordering blocked
* status = maintenance

OFF:

* normal status calculation resumes
