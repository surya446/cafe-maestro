PROJECT CONTEXT

This is a Cafe Management Platform with:

* Public customer website
* QR table ordering
* Staff dashboard
* Kitchen dashboard
* Admin dashboard
* Supabase backend

CURRENT STATUS

QR ordering system works.

Architecture:

* Each customer scans the same table QR.
* Each scan creates a separate session.
* Customer enters their name before ordering.
* Multiple customers at the same table can order independently.
* Customers can see all orders placed by the current table group.
* Orders are attributed to customer names.
* Bill is table-scoped.
* Staff can end individual sessions.
* Table groups were implemented to separate different customer groups across the day.
* Staff clears a table after customers leave.
* Next customers become a new table group.
* Old orders must never appear in a new group.

Database already contains:

* table_groups
* table_sessions.group_id
* create_session()
* get_table_orders()
* clear_table()
* staff_request_bill()

STAFF MANAGEMENT STATUS

Current system uses invitation-based onboarding.

Problems encountered:

* Supabase invite rate limits
* Broken invite links
* Redirect issues
* Expired invitation issues
* User cleanup issues

DECISION

Invitation-based onboarding must be completely removed.

NEW REQUIREMENTS

Replace invitation onboarding with direct account creation.

NEW FLOW

Owner/Admin
↓
Create Staff Account
↓
Enter:

* Full Name
* Email
* Role
  ↓
  System generates secure temporary password
  ↓
  Create Supabase Auth user immediately
  ↓
  Create/update staff_users record
  ↓
  Send email containing:
* Email
* Temporary password
* Login URL
  ↓
  Staff logs in
  ↓
  Forced password change
  ↓
  Normal access
