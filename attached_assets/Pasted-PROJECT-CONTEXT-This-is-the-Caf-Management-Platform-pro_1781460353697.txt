We are now replacing that architecture.

FINAL DECISIONS (LOCKED)

Session Model:

* Every QR scan creates a NEW customer session.
* Customer must enter a name before ordering.
* Multiple active sessions can exist simultaneously on the same table.
* Orders belong to a session.
* Customers can see ALL orders placed on the table.
* Staff can see which customer/session placed each order.
* Staff can end one session.
* Staff can end all sessions on a table.
* Historical sessions must remain in the database.

Billing Model:

* Bill is PER TABLE.
* One combined table bill.
* Request Bill is table-scoped, not session-scoped.

Audit Data:

Store:

* customer_name
* session_id
* device_token
* user_agent
* started_at
* ended_at

Name is for display only, not identity verification.
